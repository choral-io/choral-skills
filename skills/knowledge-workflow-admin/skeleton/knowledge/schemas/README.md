# Knowledge Schemas

This directory contains the writing contracts for the repository knowledge base.

Read `common.md` before writing or reorganizing shared knowledge. Then read the schema that matches the target area:

| Area                | Schema            |
| ------------------- | ----------------- |
| `discovery/`        | `discovery.md`    |
| `product/`          | `product.md`      |
| `design/`           | `design.md`       |
| `architecture/`     | `architecture.md` |
| `concepts/`         | `concepts.md`     |
| `decisions/`        | `decisions.md`    |
| `guidelines/`       | `guidelines.md`   |
| `planning/`         | `planning.md`     |
| `planning/sprints/` | `sprints.md`      |
| `proposals/`        | `proposals.md`    |
| `tasks/`            | `tasks.md`        |
| `members/`          | `members.md`      |
| `groups/`           | `groups.md`       |
| `workspace/`        | `workspace.md`    |

Schemas are workflow rules, not business facts. They may appear in Foam backlinks and graph views, but delivery planning and task selection must not treat schema files as product, design, architecture, decision, or task inputs.
